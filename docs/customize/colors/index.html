Colors

Hummingbird provide a small set of semantic color variables, which makes it easy to apply and customize consistently across your project.

Overview

Instead of applying fixed Tailwind palette classes like:

  • text-red-500
  • bg-green-600
  • border-blue-700 etc.

It’s better to rely on semantic color classes. Semantic colors are mapped to your design tokens and automatically adapt when you change themes.

For example, you can use:

  • text-subtle
  • bg-secondary
  • border-light etc.

This way, you’re not tied to a specific palette value (blue-500, green-600, etc.). Instead, your components reference semantic roles (primary, secondary, warning), which makes your UI easier to maintain, brand, and theme.

Behind the scenes, each semantic color class points to a CSS variable. Updating or overriding that variable updates the color everywhere it’s used.

How to use

Now you can use semantic color classes like:

  • bg-primary
  • text-muted
  • border-base

These classes use your theme’s color variables, so your UI stays consistent when you customize or switch themes.

<button class="bg-primary text-contrast hover:bg-primary-dark">
  Button
</button>

<p class="text-muted">Muted paragraph text</p>

<div class="border border-base bg-default p-4">
  Card with subtle border
</div>
Customize color

You can override any color by redefining its CSS variable inside a @theme block after you import Hummingbird UI:

@import "tailwindcss";
@import "@hummingbirdui/hummingbird";

@theme {
--background-color-subtle: #eff2f8;
--color-primary: #0074e6;
--color-primary-dark: #001c38;
}
