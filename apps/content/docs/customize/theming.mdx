##### Overview

Hummingbird UI is built on Tailwind CSS v4 and exposes a set of theme variables (design tokens) for colors, typography, shadows, and more. You can:

- Use our theme variables (like `--color-primary` and `--shadow-md`) via standard Tailwind utilities (for example `bg-primary`, `text-primary`, `shadow-md`).

- Override any theme variables globally by redefining it in an `@theme { ... }` block after importing our CSS.

- Create multiple themes (e.g. light/dark/forest) by overriding the theme variables within a theme scope like `@variant dark{}`, or `[data-theme="dark"]{}`.

###### Overriding Variables

Redefine any theme variables by declaring it in `@theme` after you import Hummingbird UI:

```css
@import "tailwindcss";
@import "@hummingbirdui/hummingbird";

@theme {
--color-primary: #0074e6;
--font-sans: "Figtree", ui-sans-serif, system-ui, sans-serif;
}
```

###### Multiple themes

We use a small set of theme variables. By overriding these variables inside a new theme scope, you can create multiple themes.

```sass
@theme {
  --color-primary: var(--color-blue-500);
  --color-secondary: var(--color-purple-500);
}

@variant dark {
  --color-primary: var(--color-blue-400);
  --color-secondary: var(--color-purple-400);
}

@variant forest {
  --color-primary: var(--color-blue-600);
  --color-secondary: var(--color-purple-600);
}
```
