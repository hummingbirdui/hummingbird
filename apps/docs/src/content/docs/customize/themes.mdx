## Getting Started

Only one theme should be imported at a time, as importing multiple themes will cause the last one to override the others. The `default` theme acts as a fallback if no specific theme is imported.

<Example lang="css" showPreview={false} 
  code={`/* Import the desired theme */
  @import "@hummingbirdui/hummingbird/src/themes/luxury.css";

  /* @import "@hummingbirdui/hummingbird/src/themes/retro.css"; */
  /* @import "@hummingbirdui/hummingbird/src/themes/arctic.css"; */
  /* @import "@hummingbirdui/hummingbird/src/themes/nature.css"; */
  /* @import "@hummingbirdui/hummingbird/src/themes/dracula.css"; */
  /* @import "@hummingbirdui/hummingbird/src/themes/ember.css"; */
  /* @import "@hummingbirdui/hummingbird/src/themes/midnight.css"; */
`} />

## Customizing Themes

Hummingbird's themes are built using CSS variables, making them incredibly easy to customize. Specific tokens can be overridden globally to change the look and feel of an entire application.

### Updating Tokens

To update a token, redefine the CSS variable within the `@theme` block for the specific theme attribute. For example, to change the primary color:

<Example lang="css" showPreview={false} 
  code={`@theme {
    /* Override the primary color tokens */
    --color-primary-lighter: #e0e7ff;
    --color-primary-light: #818cf8;
    --color-primary: #6366f1;
    --color-primary-dark: #4f46e5;
    --color-primary-darker: #312e81;
}

  /* Update the dark mode tokens */
  @layer theme {
    :root, :host {
      @variant dark {
        --color-primary-lighter: #312e81;
        --color-primary-light: #4f46e5;
        --color-primary: #6366f1;
        --color-primary-dark: #818cf8;
        --color-primary-darker: #e0e7ff;
      }
    }
  }
`} />

By overriding these variables, all components that rely on the `--color-primary` token will automatically update to reflect the new custom colors.

## Token Reference

A complete set of semantic color tokens is available for customization, providing consistent control over the global color scheme, component states, and interface styling throughout the application.

For a comprehensive list of all available color tokens, their structures, and how to use them, please refer to the [Colors documentation](/docs/customize/colors).
